# OBSBOT Admin Dashboard

A full-stack Node.js + React control interface for managing multiple **OBSBOT Tail 2** cameras across a network.

---

## Features

- **Camera Management** — Add, edit, and remove cameras by IP address.
- **Group Assignment** — Organize cameras into groups for batch operations.
- **Start/Stop Auto Tracking** — Toggle tracking mode on a single camera or an entire group.
- **PTZ Joystick** — Pan/tilt/zoom control with roll and speed sliders per camera.
- **Preset Reset** — Send cameras back to Preset 1 individually.
- **Gesture Control Toggle** — Disable gesture controls on any camera.
- **Batch Command Feedback** — Warning if some cameras fail during a batch command.
- **Dark Mode UI** — Built with TailwindCSS and Radix UI.
- **Log System** — Timestamped terminal logs with color-coded severity via `chalk`.
- **Check Alive** — Manually verify network reachability for each camera.
- **Built-in Updater** — Check for new releases and apply updates from the UI.

## Installation

### Option A — Windows `.exe` (no Node.js required)

Download the latest `obsbot.exe` from the [Releases](https://github.com/x3rud/Obsbot-manager/releases) page.  
Double-click it — the server starts and the dashboard opens in your browser automatically.  
The database (`obsbot.db`) is created in the same folder as the exe.

---

### Option B — Install script (macOS / Linux)

```bash
curl -fsSL https://raw.githubusercontent.com/x3rud/Obsbot-manager/main/install.sh | bash
cd ~/.obsbot
pnpm start
```

Re-run the script at any time to update to the latest version.

---

### Option C — Install script (Windows, PowerShell)

```powershell
irm https://raw.githubusercontent.com/x3rud/Obsbot-manager/main/install.ps1 | iex
cd "$HOME\.obsbot"
pnpm start
```

---

### Option D — Manual (git clone)

```bash
git clone https://github.com/x3rud/Obsbot-manager.git
cd Obsbot-manager
pnpm install --force
pnpm run build       # builds the React frontend
pnpm start           # starts the server on http://localhost:3001
```

Then open [http://localhost:3001](http://localhost:3001).

## Updating

### From the UI

Click **Check updates** in the top-right corner of the dashboard.  
If a new version is available a yellow **Update to vX.X.X** button appears.  
Clicking it runs `git pull → pnpm install → pnpm run build` on the server, then prompts you to restart.

> Requires the app to have been installed via git clone (Options B–D). The Windows `.exe` can be replaced by downloading the new release.

### Manual

```bash
cd ~/.obsbot        # or your install directory
git pull
pnpm install --force
pnpm run build
pnpm start
```

## Camera API Requirements

Each OBSBOT camera must expose a reachable REST API on its local IP (e.g. `http://192.168.1.100`).

Camera status detection uses **puppeteer-core** with the system browser (Edge on Windows, Chrome on macOS/Linux).  
If no browser is found automatically, set the `PUPPETEER_EXECUTABLE_PATH` environment variable to your browser's executable path.

## Building the Windows exe locally

```bash
pnpm run build:exe
# → dist/obsbot.exe
```

To use a custom icon, place a `.ico` file at `assets/icon.ico` and add `--icon assets/icon.ico` to the `build:exe` script.

The GitHub Actions workflow (`.github/workflows/release.yml`) builds and attaches `obsbot.exe` to every release automatically when you push a version tag:

```bash
git tag v0.2.0
git push --tags
```
